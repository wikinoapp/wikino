## 概要

<!--

`internal/templates/helper.go` の `Icon()` 関数内にアイコンのSVGデータがすべてフラットに格納されており、アイコン数の増加に伴いファイルが肥大化している。

アイコンは主に [Phosphor Icons](https://phosphoricons.com/) から取得しているが、`logo`（Wikino独自）や `caret-right` など別の出典のアイコンも含まれている。現状ではどのアイコンがどのライブラリに由来するか判別できず、著作者情報も記載されていない。

アイコンのSVGデータを出典ライブラリごとに別ファイルに分離し、各ファイルに著作者情報をコメントとして記載することで、ライセンス管理と保守性を向上させる。

### icons_custom.go

Phosphor Icons 以外のアイコンを格納する。各アイコンの出典情報をコメントで記載する。

```go
package templates

// カスタムアイコン（Phosphor Icons 以外のアイコン）

var customIcons = map[string]string{
    // Wikino ロゴ
    "logo": `<svg ...>`,

    // caret-right
    "caret-right": `<svg ...>`,
}
```

### Icon() 関数の変更

`helper.go` の `Icon()` 関数は、関数内のインラインマップを削除し、`phosphorIcons` と `customIcons` の両方から検索するように変更する。

```go
func Icon(name string, class ...string) templ.Component {
    // 各マップから検索
    svg, ok := phosphorIcons[name]
    if !ok {
        svg, ok = customIcons[name]
    }
    if !ok {
        // デフォルトとしてinfoアイコンを使用
        svg = phosphorIcons["info-regular"]
    }

    // クラス名が指定されている場合は、SVGタグに追加
    if len(class) > 0 && class[0] != "" {
        svg = `<svg class="` + class[0] + `" ` + svg[5:]
    }

    return templ.Raw(svg)
}
```

### 名前空間を第1引数に追加する

`Icon("phosphor", "arrow-left", "w-4")` のように名前空間を第1引数に追加する方式を検討した。

**不採用の理由**:

- アイコンの大半は Phosphor Icons であり、毎回 `"phosphor"` を指定するのは冗長
- 全呼び出し箇所（約30箇所の .templ ファイル）の修正が必要になる
- 名前衝突のリスクが低く、名前空間による切り替えが実際に必要になる場面がない
- YAGNI原則に反する

### コロン区切りの名前空間

`Icon("phosphor:arrow-left", "w-4")` のようにコロン区切りで名前空間を指定する方式を検討した。

**不採用の理由**:

- 名前空間を第1引数にする方式と同様の冗長性の問題がある
- 文字列の規約に依存するため型安全性が低い
- ファイルレベルの分離で著作者管理の目的は達成できる
