## 概要

<!--
ガイドライン:

トピック詳細画面（`GET /s/:space_identifier/topics/:topic_number`）を Rails 実装から Go 実装に移行する。トピック詳細画面は、トピック内のページを一覧表示する画面であり、ピン留めページと通常ページをグリッド表示し、カーソルベースのページネーションを提供する。

**目的**:

- 編集提案機能（[edit-suggestion.md](edit-suggestion.md)）では、トピック詳細画面に「ページ」と「編集提案」のタブを追加する必要がある。Go 版で先にトピック詳細画面を実装しておくことで、編集提案機能の実装をスムーズに行える
- Rails から Go への段階的移行の一環として、トピック関連の表示機能を Go に移す

**現状**:

- Go 版では Domain/Infrastructure 層（Model、Repository、Query）は実装済み
- Presentation 層（Handler、ViewModel の詳細画面用、Template）が未実装
- 現在はリバースプロキシで Rails 版にフォールバックされている

### 機能要件

<!--
「ユーザーは〇〇できる」「システムは〇〇する」という形式で記述

-->

- ユーザーはトピック詳細画面（`/s/:space_identifier/topics/:topic_number`）にアクセスし、トピック内のページ一覧を表示できる
- ピン留めされたページは画面上部にグリッド表示される
- 通常ページはピン留めページの下にグリッド表示され、更新日時の降順でソートされる
- ページが 100 件を超える場合はオフセットベースのページネーションで移動できる
- トピックメンバーは「新規ページ作成」ボタンからページ作成画面に遷移できる
- トピック管理者はトピック設定画面へのリンクにアクセスできる
- 非公開トピックにアクセス権のないユーザーがアクセスした場合は 404 を返す
- 各ページカードにはタイトル、アイキャッチ画像（存在する場合）、ピン留めアイコン（ピン留めの場合）が表示される

### 既存実装の活用

Go 版では以下が既に実装済みであり、そのまま活用する:

- **Model**: `model.Topic`、`model.TopicMember`、`model.Page`
- **Repository**: `TopicRepository`（`FindBySpaceAndNumber`、`ListActiveBySpace` 等）、`TopicMemberRepository`、`PageRepository`
- **Query**: topics、topic_members、pages の SQL クエリ
- **ViewModel**: `viewmodel.Topic`（基本情報）、`viewmodel.CardLinkPage`（ページカード）

### エンドポイント

| HTTP メソッド | URL パターン                                | リソースディレクトリ | ファイル  | メソッド | 説明             |
| ------------- | ------------------------------------------- | -------------------- | --------- | -------- | ---------------- |
| GET           | `/s/:space_identifier/topics/:topic_number` | `topic/`             | `show.go` | `Show`   | トピック詳細画面 |

### ハンドラー設計

`internal/handler/topic/` ディレクトリに以下のファイルを配置:

- `handler.go` - Handler 構造体と依存性
- `show.go` - トピック詳細画面の表示

Handler の依存性:

```go
type Handler struct {
    cfg             *config.Config
    spaceRepo       *repository.SpaceRepository
    spaceMemberRepo *repository.SpaceMemberRepository
    topicRepo       *repository.TopicRepository
    topicMemberRepo *repository.TopicMemberRepository
    pageRepo        *repository.PageRepository
    sessionMgr      *session.Manager
}
```

### 処理フロー（Show）

1. URL パラメータから `space_identifier` と `topic_number` を取得
2. `SpaceRepository` でスペースを取得
3. ログインユーザーの `SpaceMember` を取得（未ログインなら nil）
4. `TopicRepository` でトピックを取得
5. 権限チェック: トピックが非公開の場合、トピックメンバーでなければ 404
6. ピン留めページを取得（`pinned_at IS NOT NULL`、`pinned_at DESC` でソート）
7. 通常ページをオフセットベースページネーションで取得（`modified_at DESC`、100 件）
8. ViewModel に変換してテンプレートをレンダリング

### ViewModel 設計

トピック詳細画面用の ViewModel を追加:

```go
// viewmodel/topic.go に追加
type TopicForShow struct {
    Name          string
    Number        int32
    Description   string
    IconName      IconName
    CanUpdate     bool
    CanCreatePage bool
    Space         SpaceForBreadcrumb
}
```

ページカード用には既存の `viewmodel.CardLinkPage` をそのまま利用する。`CardImageURL` フィールドとテンプレート側の表示ロジックは既に存在しており、`NewCardLinkPage` コンストラクタでアイキャッチ画像 URL を設定する処理を追加するだけで対応できる。

### オフセットベースページネーション

Rails 版ではカーソルベースページネーション（`cursor_paginate` gem）を使用しているが、Go 版ではオフセットベースページネーションを採用する。

- `page` パラメータ: ページ番号（デフォルト: 1）
- `limit`: 100 件
- `modified_at DESC, id DESC` でソート

- `show.templ` - トピック詳細画面

既存のコンポーネントを活用:

- レイアウト: `layouts/default.templ`（サイドバー付き 1 カラムレイアウト）

- ページネーション: 新規作成が必要
- 空状態: 新規作成が必要

### ボトムナビコンポーネント

Rails 版の `navbars/bottom_component` に相当するモバイル用ボトムナビゲーションを Go 版で新規作成する。デスクトップでは非表示（`md:hidden`）、モバイルでのみ表示される。

表示項目:

- **メニューボタン**: サイドバーの開閉を切り替え（`basecoat:sidebar` カスタムイベント）
- **ホームボタン**: ホーム画面へのリンク（アクティブ状態でアイコン変化）
- **検索ボタン**（ログイン時のみ）: 検索画面へのリンク（アクティブ状態でアイコン変化）
- **ログインボタン**（未ログイン時のみ）: サインイン画面へのリンク

既存の `templates/page_name.go` の `PageName` 定数を使ってアクティブ状態を判定する。

### 権限チェック

Rails 版の `TopicPolicyFactory` に相当する権限チェックをハンドラー内で実装:

- **トピック表示**: 公開トピックは全員閲覧可、非公開トピックはトピックメンバーのみ
- **ページ作成**: トピックメンバーのみ
- **トピック更新**: トピック管理者（Admin ロール）またはスペースオーナーのみ

### リバースプロキシの更新

`internal/middleware/reverse_proxy.go` のホワイトリストにトピック詳細画面のパスパターンを追加し、Go 版で処理されるようにする。

### ページの SQL クエリ追加

トピック詳細画面用に以下の SQL クエリを追加する必要がある:

- ピン留めページの取得（`space_id` + `topic_id` + `pinned_at IS NOT NULL` + 公開済み + 未廃棄）
- 通常ページのオフセットベースページネーション取得（`space_id` + `topic_id` + `pinned_at IS NULL` + 公開済み + 未廃棄 + `modified_at DESC` + LIMIT/OFFSET）

### i18n

以下の翻訳キーを追加:

- `topic_show_title` - ページタイトル（`{topic_name} | {space_name}`）
- `topic_no_pages_message` - ページがない場合のメッセージ
- `topic_no_pages_description` - ページ作成を促す説明文
- `pagination_previous` - 前のページ
- `pagination_next` - 次のページ

### カーソルベースページネーション

Rails 版ではカーソルベースページネーション（`cursor_paginate` gem、`(modified_at, id)` の組み合わせを Base64 エンコード）を使用している。Go 版ではオフセットベースページネーションを採用した。

**不採用の理由**:

- トピック内のページ一覧は頻繁にデータが挿入・削除される性質ではなく、オフセットベースでも実用上の問題が少ない
- オフセットベースの方が実装がシンプルで、ページ番号による直感的なナビゲーションが可能
- 将来的にカーソルベースが必要になった場合は、ページネーションの実装を差し替えれば対応できる
