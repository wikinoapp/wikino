# 検索機能実装タスク

## 概要

全スペースのページを検索できる機能を実装します。参加していないスペースのページも公開トピック内のページであれば検索可能です（GitHubの検索機能と同様）。

## 要件

- URL: `GET /search?q=キーワード`
- 検索対象: 
  - ユーザーが参加しているスペース内の全ページ
  - 参加していないスペースの公開トピック内のページ
- 検索方法: ページタイトルの部分一致検索（ILIKE）
- UI: キーワード入力欄と検索結果のページリンク表示

## 実装の詳細

### データベースクエリ

#### 通常の検索（space:フィルターなし）

```ruby
# 参加しているスペースのページ
member_pages = PageRecord
  .joins(space_record: :space_member_records)
  .where(space_member_records: {user_id: current_user_record!.id})
  .where("pages.title ILIKE ?", "%#{keyword}%")
  .active

# 公開トピックのページ（参加していないスペースも含む）
public_pages = PageRecord
  .joins(:topic_record)
  .where(topic_records: {visibility: TopicVisibility::Public.serialize})
  .where("pages.title ILIKE ?", "%#{keyword}%")
  .active

# 結果を結合して重複を除去
PageRecord
  .where(id: member_pages.select(:id).union(public_pages.select(:id)))
  .order(modified_at: :desc)
  .limit(50)
```

#### space:フィルター付き検索

```ruby
# 参加しているスペースのページ
member_pages = PageRecord
  .joins(:space_record)
  .joins("INNER JOIN space_members ON spaces.id = space_members.space_id")
  .where("space_members.user_id = ? AND space_members.active = ?", current_user_record!.id, true)
  .where("spaces.identifier IN (?)", space_identifiers)
  .active

# 指定されたスペースの公開トピックのページ
public_pages = PageRecord
  .joins(:space_record, :topic_record)
  .where(spaces: {identifier: space_identifiers})
  .where(topic_records: {visibility: TopicVisibility::Public.serialize})
  .active

# 結果を結合
PageRecord
  .where(id: member_pages.select(:id).union(public_pages.select(:id)))
  .order(modified_at: :desc)
  .limit(50)
```

- 参加スペースの全ページと公開トピックのページを両方検索
- UNIONを使用して結果を結合し、重複を除去
- `ILIKE`でタイトルの部分一致検索（大文字小文字区別なし）
- 結果は50件に制限し、最近編集した順（modified_at降順）でソート

### データ変換

- `PageRepository#to_models`を使用してPageRecordからPageモデルに変換
- ビューには`T::Array[Page]`で渡す
- Pageモデルの`display_title`メソッドを使用（無題の場合の処理含む）

### UI/UX

- 検索フォームは`GET`メソッドを使用
- 検索パラメータ: `q`（一般的な検索パラメータ名）
- 検索結果は即座に表示（ページリロード）
- 検索結果がない場合は「検索結果が見つかりませんでした」と表示
- 検索結果は件数表示付きでカード形式で表示
- 各結果にはページタイトル、スペース/トピック名、最終更新日時を表示

### セキュリティ

- 参加しているスペースの全ページと、公開トピック内のページのみを検索
- プライベートトピック内のページは参加者のみ検索可能
- SQLインジェクション対策（プレースホルダーの使用）
- XSS対策（ビューでの適切なエスケープ）
- 検索キーワードの形式チェック（`<>`などの不正文字を拒否）

### フォームバリデーション

- 最小文字数: 2文字以上
- 最大文字数: 100文字以下
- 禁止文字: `<`, `>`
- 空文字列・nilの場合は検索を実行しない
- **バリデーションメッセージのI18n対応完了**: エラーメッセージは`config/locales/forms.ja.yml`で管理

### 型安全性

- Sorbetによる型チェック対応
- 全メソッドに適切な型注釈を付与
- PageRecordからPageモデルへの変換で型安全性を確保
